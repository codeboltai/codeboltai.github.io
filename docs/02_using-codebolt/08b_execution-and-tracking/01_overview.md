# Execution & Tracking

When agents run, Codebolt gives you a set of panels to monitor what's happening, configure where it happens, and manage long-running or background work. All of these are accessible from the **Execution** dropdown in the bottom bar.

## Panels in this section

| Panel | Bottom bar path | Purpose |
|---|---|---|
| [Thread Panel](./02_thread-panel.md) | Execution → Thread Panel | All message threads in this project |
| [Environments](./03_environments.md) | Execution → Environment | Configure execution environments |
| [Running Agents](./04_running-agents.md) | Execution → Running Agent | Live view of active agent processes |
| [Background Agents](./05_background-agents.md) | Execution → Background Agents | Long-running agents without a chat thread |
| [Jobs](./06_jobs.md) | Tools → Jobs | Queued and completed background jobs |
| [Chat Canvas](./07_chat-canvas.md) | Execution → Chat Canvas | Visual canvas for multi-agent coordination |

## The Execution dropdown

Click the **Execution** label in the bottom bar to open a dropdown with two groups:

**Tracking:**
- Thread Panel
- Environment
- Running Agent
- Chat Canvas
- Background Agents

**Orchestration:**
- Auto Testing
- Orchestrator Management
- Distributed Swarm Management
- Review Merge Request
- Agent Deliberation
- Update Project Structure Request

The tracking group is for observing what's happening. The orchestration group is for setting up and managing multi-agent workflows — see [Multi-Agent Usage](../07_multi-agent-usage/01_overview.md) for those.
