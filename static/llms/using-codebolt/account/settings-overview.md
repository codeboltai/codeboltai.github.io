# Settings

The Settings panel is the central place to configure Codebolt — LLM providers, agent behaviour, environment tools, language servers, MCP servers, appearance, and more.

Open via: **Top navbar → Settings button** or **Bottom bar → System → Settings**

## Settings sections

| Section | Purpose |
|---|---|
| [Default Settings](./06_llm-settings.md#default-settings) | Choose the default LLM and embedding model for the application |
| [LLMs](./06_llm-settings.md#llm-providers) | Add and manage LLM provider API keys; browse available models |
| [Agents](./07_agent-settings.md#agent-llm-settings) | Assign LLMs to individual agents and their roles |
| [Universal Agent](./07_agent-settings.md#universal-agent) | Configure a custom or offline Universal Agent endpoint |
| [Agent Steps](./07_agent-settings.md#agent-steps) | Assign agents to each stage of the development workflow |
| [Code Editor Setting](./07_agent-settings.md#code-editor-setting) | Choose agents for Cmd+K inline editing and codemap generation |
| [Environment & Services](./08_environment-services.md#environment--services) | Verify installed runtimes and manage external service API keys |
| [Environment Provider](./08_environment-services.md#environment-provider) | Add and configure Docker, E2B, local, and remote execution providers |
| [Merge Config](./08_environment-services.md#merge-config) | Set strategy for merging environment changes back into the project |
| [Language Servers](./09_language-servers.md) | Install and configure LSP servers for code intelligence |
| [MCP Servers](./10_mcp-servers.md) | Install and manage Model Context Protocol servers |
| [Pheromones](./11_appearance.md#pheromones) | Define custom pheromone types for stigmergic agent coordination |
| [Appearance](./11_appearance.md#appearance) | Customise the application visual theme |
| [Global Settings](./11_appearance.md#global-settings) | CLI setup and editor background colour |
